End-to-end integration
tests for every branch.

Run tests on a full-stack environment every time you update a branch—complete with seeded databases, caches, service discovery, etc. Find issues during development that were only possible to find during staging or production.

Support for Selenium-based tests.

First-class support for Selenium/Webdriver tests is included. Run comprehensive tests automatically after a branch update, and spot regressions by reviewing screenshots and videos.

Automated test environment management.

Environments are automatically created when a new branch is created, and they’re discarded when the branch is deleted or becomes inactive. Caching keeps this fast by reducing the number of builds needed, and new builds always have at least 1 CPU core.

Notifications on pull requests & Slack.

Give visibility to your entire team right where they need it. Integration test results are displayed on the Pull Requests page, along with the environment URL to speed up reviews. Enable Slack integration to be instantly notified when a test passes or fails.

Docker Compose support.

Runnable uses native Docker components under the hood (Compose, Registry, Swarm), so it’s easy to pull in Dockerfiles and a docker-compose.yml file to get started right away. Running full Docker eliminates the risk of vendor lock-in. You’re always able to export your assets.

How is this different from
Jenkins CI & Cloudbees?

With enough configuration, expertise, and maintenance resources, it’s possible to build a similar workflow in-house. We’ve found that the majority of our users have encountered quite a few roadblocks and quickly learn that it’s more cost-effective to use a reliable service that works well with their workflow, has no vendor no lock-in (full Docker support), and includes infrastructure costs.

How is this different from
CircleCI or Travis CI?

Runnable enables end-to-end testing on top of automated unit and functional test running and building of artifacts. Runnable also charges per developer, not per resources used at any given time.

How is this different from
Sauce Labs?

Runnable makes it easy to support many forms of testing by always having a complete, isolated environment to test against with every code change. This enables us to support running and reporting Selenium or Webdriver tests.

How is this different from
Docker Cloud or Tutum?

Docker Cloud is a hosted service for docker containers. While Runnable also hosts containers, it has been built from the ground up to make end-to-end testing as easy as possible. Runnable also charges per developer, not per resources used at any given time.

Q: Do I need to use Docker?

No. Our setup process was built to easily onboard applications regardless of whether they are Dockerized.

Q: How secure is my code?

Your code is only transferred to an isolated set of servers that are not accessible to any other users. This is commonly referred to as single-tenant infrastructure. Runnable also undergoes regular security audits to ensure your code stays secure.

Q: What does the setup process look like?

For many teams, it’s as easy as selecting your repositories, adding in environment variables or replace rules (for hard-coded values), and then saving. Adding databases and other services is also point-and-click. You’ll be invited to join our engineering team’s Slack, so you’ll have direct access to our entire team to help with setup. We can even set up your app for you. :)
